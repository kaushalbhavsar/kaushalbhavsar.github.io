What I treat as real problems

The hardest threats aren’t loud. They look like legitimate work until you notice drift, timing, and access paths.

  • Privilege creep and slow access expansion
  • Data exposure spread across many small “reasonable” actions
  • Misuse hidden inside normal workflows
  • Anomalies that need context, not thresholds

How I keep it usable

I optimize for investigator decision quality. Fewer findings, higher confidence, clear explanations.

  • make “why” visible, not just “what”
  • link anomalies to supporting signals
  • design for missing data and imperfect telemetry
  • ship systems that remain trustworthy at scale
My definition of a useful detection: it changes what an investigator would do next — and it can defend that recommendation.

How I judge signal quality

  • they survive noisy enterprise reality
  • they capture sequences and change over time
  • they can be explained to a human
  • they reduce uncertainty, not add fear

What I don’t build

  • dashboards that re-label alerts
  • AI wrappers with no grounding
  • fear-first security marketing
  • checklist products
Most security asks: “Did this violate a rule?”
I ask: “Does this behavior still make sense — given who, when, and what changed?”
What I build
I describe work as systems. Each lane below is something I can design end-to-end: data → logic → output → workflow.
Systems > Titles
Behavior baselines that survive reality
Time-aware modeling

Not “static user profiles.” I model normal as a moving target: seasonality, role shifts, project context, and drift.

  • baseline per user / team / resource
  • drift detection: slow changes that look harmless
  • rare paths: “unusual but plausible” access
Signals you can defend in a review
Explainable decisions

If the system can’t explain itself, it doesn’t deserve production trust.

  • evidence bundles for each alert
  • counterfactual checks: what would make this normal?
  • reason codes: why this is worth attention
Investigator workflows, not alert floods
Action-first UX

“Next best action” beats “top 100 alerts.” I design systems that guide decisions.

  • triage views that collapse noise
  • link anomalies into one story
  • human-in-the-loop escalation thresholds
Privacy-safe analytics for security
Sensitive data aware

I like systems that can learn from data without exposing it. Security shouldn’t require oversharing.

  • anonymization / redaction-aware pipelines
  • least-privilege telemetry collection
  • safe aggregation strategies
My opinion: The future of detection isn’t “more AI.” It’s better decision systems: fewer, stronger, explainable calls.
